## Project Purpose

This repository is a personal knowledge organization system for computer science topics. The goal is to create a structured, interconnected learning framework that makes complex CS concepts easy to understand and navigate.

**Core Principles:**
- **Easy to read**: Content should be clear, well-formatted, and accessible
- **Well-structured**: Organize knowledge hierarchically and logically
- **Internally related**: Create meaningful connections between related topics
- **Study-focused**: Optimize for learning and retention, not just reference

## Current State

This project is in its early stages. The structure and approach are still being developed. When working on this repository:

1. **Prioritize clarity over completeness** - better to have well-explained foundational topics than comprehensive but unclear coverage

2. **Build connections** - when adding new content, explicitly link to related concepts already in the knowledge base

3. **Maintain consistency** - follow established formatting and organizational patterns once they emerge

4. **Think like a learner** - structure content in the order someone would naturally learn it, not just alphabetically or by category

## Working with This Repository

Since the organizational structure is evolving, be flexible and creative when the user asks to add new knowledge. Suggest organizational approaches that align with the core principles above.

When the user wants to add new computer science knowledge:
- Ask clarifying questions about depth and breadth desired
- Propose a logical place in the knowledge structure
- Create clear hierarchies and cross-references
- Use consistent formatting (markdown, diagrams, code examples as appropriate)
